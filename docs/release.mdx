# Release Automation

This repo uses Release Please for independent Dart package releases.

Release configuration lives in:

- `release-please-config.json`
- `.release-please-manifest.json`

Release tags use `<package>-v<version>`:

- `ecsly-v0.0.1-dev.x`
- `ecsly_app-v0.1.0-dev.x`
- `ecsly_codegen-v0.1.0-dev.x`
- `ecsly_flutter-v0.1.0-dev.x`

The tag workflow validates the package/version pair before publishing. Local
maintainer preflight:

```sh
dart tool/release/pub_package.dart --all --skip-existing
dart tool/release/pub_package.dart --tag ecsly-v0.0.1-dev.11 --skip-existing
```

Actual pub.dev publishing also requires the pub.dev package admin settings to
allow GitHub Actions publishing for repository `Arenukvern/ecsly` and matching
tag patterns.
